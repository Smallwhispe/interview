# Chinese（中文规范）

## 1. 执行标准

-   **核心原则**：原则上，TikTok 相关业务的代码（包括代码中的字符串、注释、日志等）中**不建议出现中文字段**。
-   **豁免场景**：
    -   多语言翻译（i18n）文件中作为语言资源存在。
    -   插件自动生成且不会在 UI 中体现的内容。
    -   测试用例（Test Case），但前提是必须提供对应的英文版本（双语），且确保相关代码不会被打包编译到最终上线的产物中。

## 2. 处理方法

-   **主要操作**：将代码或注释中的中文字符**删除**或**替换为对应的英文**。
-   **验证方式**：在完成代码修改并合入后，可以访问神盾平台，找到对应仓库的“模块管理”页面，对相关的分支（Branch）或提交（Commit）发起**手动扫描**。扫描通过后，问题即视为解决，无需关注神盾工单的状态。

## 3. 常见问题与误报处理

-   **日文被误判为中文**：
    -   由于部分日文汉字与中文汉字字符集重叠，扫描工具可能存在误判。
    -   **处理路径**：如果确认是日文被误报，可以通过 **Oncall** 通道联系神盾运营同学进行处理。
-   **确认误报后的操作**：
    -   在神盾工单上直接将问题标记为“误报”（False Positive）。
    -   工单将流转至运营同学进行二次确认。一旦确认，同一文件的同一规则误报后续将不会再次触发告警。
    -   历史检测记录可在工单详情页的下拉菜单中查看。
